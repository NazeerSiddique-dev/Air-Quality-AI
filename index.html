Exploratory Data Analysis

Data was loaded, cleaned, missing values imputed, and a rush-hour feature engineered from timestamps.

–200 used as missing marker, replaced with NaN then imputed with column means.

Theory Assignments

Deep dives into the mathematical foundations and algorithmic architectures of distributed data systems.

Assignment 1: Distributed Data Analytics & Applications

+

This investigation explores the architectural paradigm shift from centralized to decentralized data processing. We analyze how massive datasets are partitioned using techniques like Sharding and Replication to ensure both performance and fault tolerance in high-throughput environments.

Key Focus Areas:

  • Data Partitioning: Horizontal scaling strategies for elastic data growth.
  • Parallel Processing: Leveraging MapReduce and Spark core concepts.
  • Consistency Models: Balancing the CAP theorem in distributed systems.

Assignment 2: Algorithms in Distributed Analytics

+

A deep dive into advanced synchronization algorithms and communication-efficient learning. This work proposes the AGCEA (Adaptive Gradient-Compressed Edge Analytics) framework, designed to optimize bandwidth while maintaining high model accuracy in edge-computing scenarios.

Core Contributions:

  • AGCEA Framework: Novel gradient compression for federated nodes.
  • Convergence Analysis: Mathematical proofs for decentralized stochastic optimization.
  • Communication Efficiency: Minimizing round-trips in high-latency networks.
